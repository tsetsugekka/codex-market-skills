# CN Theme Strength MX

`cn-theme-strength-mx` 是一个用于 Codex 的 A 股题材涨跌幅 / 题材强弱榜 skill。它读取已有 `/themes` 股票-题材映射，用东方财富妙想 `mx-zixuan` 先抓用户自选股行情，再用 `mx-xuangu` 按代码批量补抓自选接口没有返回的题材成分股，最后按 `/themes` 权重计算题材加权涨跌幅。

它的特点是盘中也可以使用，并且会在抓取过程中报告进度：自选接口返回多少行、覆盖 `/themes` A 股多少个代码、还要补抓多少个代码、补抓批次完成到第几批、是否遇到 `操作过于频繁` 以及最终是否补齐。

skill 目录里的 `SKILL.md` 是给 Codex 执行时读取的规则文件；本文档是给 GitHub 用户阅读的公开说明。

## 能做什么

- 从 `/themes` 映射中提取 A 股股票-题材关系和权重。
- 使用 `mx-zixuan` 查询东方财富自选股列表中的最新价、涨跌幅等行情字段。
- 对 `mx-zixuan` 没有返回的 `/themes` 代码，用 `mx-xuangu` 按最多 50 个代码一批补抓。
- 遇到妙想 `操作过于频繁` 时限速等待并重试，而不是直接放弃。
- 按 `/themes` 逻辑计算题材加权涨跌幅：`sum(weight * 涨跌幅) / sum(weight)`。
- 输出 A 股题材 TOP10 和最差 10，并列出主要贡献或主要拖累个股。
- 默认不写文件，只在回答中显示结果。

## 典型请求

```text
看一下 A 股现在哪个题材涨得最好，哪个最差。
```

```text
用 /themes 的题材映射和 mx 抓一次 A 股题材强弱。
```

```text
盘中查一下我的 A 股题材涨跌幅 TOP10 和最差10。
```

## 需要的权限和来源

- 必须依赖东方财富妙想 `mx-zixuan` 和 `mx-xuangu`。
- 需要用户已经配置可用的 `MX_APIKEY`。
- 需要当前工作区或用户提供的 `/themes` 映射数据，例如 `theme-data.json`，至少包含 `market`、`code`、`theme`、`weight` 字段。
- 不使用非 MX 行情源替代本 skill 的行情层。
- `mx-zixuan` 会读取用户东方财富账户下的自选股列表；本 skill 只查询，不会自动添加、删除或修改自选股。
- 默认不保存原始 API 响应、完整自选股列表或私有关注列表。

## 输出格式

默认用中文回答，先给数据日期和抓取进度，再给两张表：

```text
数据日期：2026.05.22；抓取：mx-zixuan 200行，mx-xuangu补抓4批，363/363代码齐全。
```

```text
TOP10
排名 | 题材 | 加权涨跌幅 | 主要贡献
```

```text
最差10
排名 | 题材 | 加权涨跌幅 | 主要拖累
```

默认不输出成分数；只有用户明确要求时才补充成分数、成分明细或缺失代码清单。

## 公开安全说明

这个 skill 可以放在公开仓库中，但公开仓库只保存流程和安全规则。请不要提交 `MX_APIKEY`、`.env`、东方财富账户信息、完整自选股列表、个人关注列表、运行缓存、原始 API 响应或私有 `/themes` 数据。

本 skill 输出仅用于个人研究，不构成投资建议。
