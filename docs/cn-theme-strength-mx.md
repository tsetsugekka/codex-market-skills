# CN Theme Strength MX

`cn-theme-strength-mx` 是一个用于 Codex 的 A 股题材涨跌幅 / 题材强弱榜 skill。它读取本地 A 股题材映射缓存，用东方财富妙想 `mx-zixuan` 先抓用户自选股行情，再用 `mx-xuangu` 按代码批量补抓自选接口没有返回的题材成分股，最后按映射权重计算题材加权涨跌幅。

它的特点是盘中也可以使用，并且会在抓取过程中报告进度：自选接口返回多少行、覆盖内置映射中多少个 A 股代码、还要补抓多少个代码、补抓批次完成到第几批、是否遇到 `操作过于频繁` 以及最终是否补齐。

skill 目录里的 `SKILL.md` 是给 Codex 执行时读取的规则文件；本文档是给 GitHub 用户阅读的公开说明。

正常运行时，如果本地缓存缺失或超过 7 天，skill 会尝试刷新 `theme-data.json` 和 `theme-label-i18n.json`。如果用户显式提供更新的本地题材映射文件路径，可以在当次运行使用用户提供的文件。

回答正文默认不点名映射刷新来源、候选池名称或本地路径；只写“题材映射缓存已刷新/跳过/不可用”等中性状态。用户要求来源时，可以在单独 source 列表放 URL。

## 能做什么

- 从本地映射缓存中提取 A 股股票-题材关系和权重。
- 使用 `mx-zixuan` 查询东方财富自选股列表中的最新价、涨跌幅等行情字段。
- 对 `mx-zixuan` 没有返回的映射内代码，用 `mx-xuangu` 按最多 50 个代码一批补抓。
- 遇到妙想 `操作过于频繁` 时限速等待并重试，而不是直接放弃。
- 按映射逻辑计算题材加权涨跌幅：`sum(weight * 涨跌幅) / sum(weight)`。
- 输出 A 股题材 TOP10 和 BOTTOM10，并列出主要贡献或主要拖累个股。
- 对 TOP3 题材分别选取涨幅最高的 1 只代表股，用股吧讨论线索和 `mx-search` 资讯辅助判断题材为什么异动。
- 默认不写文件，只在回答中显示结果。

## 典型请求

```text
看一下 A 股现在哪个题材涨得最好，哪个最差。
```

```text
看看当前 A 股题材强弱，显示 TOP10 和 BOTTOM10。
```

```text
盘中查一下 A 股题材涨跌幅，哪些题材最强、哪些最弱。
```

## 【依赖】

- 必需：`mx-zixuan`、`mx-xuangu`、`mx-search`，且用户已配置可用的 `MX_APIKEY`。
- 可选：`mx-data`，用于后续追查具体题材、个股、财务或估值。

## 【协同调用】

- `cn-stock-move-reason`：默认 TOP3 题材驱动检查只使用其股吧/讨论发现能力，不做完整个股异动报告。

## 本地缓存

- `assets/themes/theme-data.json`
- `assets/themes/theme-label-i18n.json`

## 需要的权限和来源

- 默认使用本地 A 股题材映射缓存：`assets/themes/theme-data.json` 至少包含 `market`、`code`、`theme`、`weight` 字段。
- 默认使用本地 `assets/themes/theme-label-i18n.json` 输出中文题材名；题材展示名优先使用其中的 `zh` 字段，只有找不到中文标签时才回退到原始题材名。
- 如果缓存缺失或超过 7 天，刷新脚本只访问 `theme-data.json` 和 `theme-label-i18n.json`；不访问题材材料文件或行情快照文件。
- 不使用非 MX 行情源替代本 skill 的行情层。
- `mx-zixuan` 会读取用户东方财富账户下的自选股列表；本 skill 只查询，不会自动添加、删除或修改自选股。
- 默认不保存原始 API 响应、完整自选股列表或私有关注列表。

## 输出格式

默认用中文回答，先给数据日期和抓取进度，再给两张表：

```text
数据日期：2026.05.22；抓取：mx-zixuan 200行，mx-xuangu补抓4批，363/363代码齐全。
```

```text
TOP10
排名 | 题材 | 加权涨跌幅 | 主要贡献
```

```text
BOTTOM10
排名 | 题材 | 加权涨跌幅 | 主要拖累
```

表格里的 `题材` 使用本地标签缓存里的中文题材名，不直接显示日文题材名。

随后输出简短的 `TOP3 题材驱动检查`。该部分只把每个 TOP3 题材里涨幅最高的 1 只代表股作为线索来源，结合股吧讨论和 `mx-search` 资讯推断题材层面的驱动；不是完整个股异动报告。

每条驱动检查默认包含：

- `代表股`：题材内涨幅最高的 1 只股票。
- `股吧线索`：市场讨论正在交易什么想象或情绪。
- `mx资讯`：妙想资讯里能支撑或否定该题材逻辑的新闻、公告、研报或政策线索。
- `题材推断`：把代表股线索上升到题材层面的解释。
- `确定度`：较高 / 中等 / 较低。若只是个股独立逻辑、旧消息、海外同情绪或股吧思惑，应降低确定度。

默认不输出成分数；只有用户明确要求时才补充成分数、成分明细或缺失代码清单。

## 公开安全说明

这个 skill 可以放在公开仓库中。请不要提交 `MX_APIKEY`、`.env`、东方财富账户信息、完整自选股列表、个人关注列表、题材映射本地缓存、运行缓存或原始 API 响应。

## 刷新题材缓存

正常使用时，如果本地缓存缺失或超过 7 天，可以刷新本地快照：

```bash
python3 <cn-theme-strength-mx>/scripts/refresh_theme_assets_from_daytrading_monster.py
```

该脚本只下载 `theme-data.json` 和 `theme-label-i18n.json`，并写入 `.last_refresh.json` 作为本地刷新记录。

手动更新本地缓存时，也可以从已确认可使用的本地目录复制文件：

```bash
python3 skills/cn-theme-strength-mx/scripts/refresh_theme_assets.py /path/to/public-theme-source
```

两个脚本都会校验基本结构。本地缓存不需要提交到公开仓库。

本 skill 输出仅用于个人研究，不构成投资建议。
