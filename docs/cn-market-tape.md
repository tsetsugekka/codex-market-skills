# CN Market Tape

`cn-market-tape` 是一个用于 Codex 的 A 股盘面分析 skill，覆盖盘中和盘后四类数据：题材强弱、板块资金流、涨停池和机构调研。

题材强弱仍使用本地 A 股题材映射缓存和 MX 成分行情，按映射权重计算题材加权涨跌幅；其他盘面模块优先使用 MX 的聚合查询，MX 不支持或字段不完整时再切换到公开聚合备用接口。

## 能做什么

- 盘中/盘后计算题材强弱 `TOP10/BOTTOM10`，输出加权涨跌幅、主要贡献或拖累股票。
- 盘中/盘后计算板块主力净流入和净流出，按 `主力净流入 Top10`、`主力净流出 Top10` 的格式输出。
- 盘中/盘后查看涨停池，统计涨停数量、连板梯队、炸板、封板率和行业/题材分布等可用字段。
- 盘中/盘后查看机构调研数据，区分最近交易日、近 7 个自然日和更长历史窗口。
- 历史请求优先读取公开历史调研热度数据；如果没有对应日期或字段，明确写“尚不支持该时间/数据”，不会用当前快照冒充历史数据。
- 题材榜默认只检查 TOP3 题材的代表股驱动；用户要求时再增加新闻、股吧、宏观或技术分析。

## 典型请求

```text
盘中看一下 A 股题材强弱，给我 TOP10 和 BOTTOM10。
```

```text
看看当前主力净流入 Top10、主力净流出 Top10，以及涨停池。
```

```text
盘后复盘题材、资金流、涨停池和机构调研，区分今日数据和历史数据。
```

```text
查某个历史日期的机构调研热度；没有数据就明确告诉我不支持。
```

## 数据口径

### 题材强弱

题材映射缓存位于：

- `assets/themes/theme-data.json`
- `assets/themes/theme-label-i18n.json`

每行至少包含 `market`、`code`、`theme`、`weight`。只保留 `market == "CN"` 且不是 `未分類` 的记录。题材加权涨跌幅计算为：

```text
sum(theme_weight * stock_chg_pct) / sum(theme_weight)
```

行情层先用 `mx-zixuan` 快速覆盖，再用 `mx-xuangu` 按最多 50 只代码一批补齐。补抓批次串行执行；遇到妙想限频或 503 时等待后重试，并在状态行报告覆盖率和缺失情况。

题材表默认格式：

```text
TOP10
排名 | 题材 | 加权涨跌幅 | 主要贡献

BOTTOM10
排名 | 题材 | 加权涨跌幅 | 主要拖累
```

### 板块资金流

先用 MX 查询板块/行业主力资金；如果 MX 不支持、字段为空、日期不匹配或接口不可用，再使用公开聚合资金流备用源。备用源按整页或整榜聚合读取，不逐板块轮询。

默认格式：

```text
数据时间：YYYY-MM-DD HH:MM；资金口径：主力净流入/净流出；来源：MX 或备用聚合源。

主力净流入 Top10
排名 | 板块 | 主力净流入

主力净流出 Top10
排名 | 板块 | 主力净流出
```

金额统一尽量换算为亿元，并标注是盘中快照还是收盘值。板块标签可能重叠，不把两张榜的金额相加，也不把不同来源的金额直接平均。

### 涨停池

优先读取当天的聚合涨停池，并校验交易日和统计时间。输出可用的：

- 涨停总数、有效普通涨停数、跌停数。
- 最高连板数、2 板及以上数量和连板梯队。
- 炸板数、封板率、封单金额或成交额。
- 行业/题材分布和与当前主线相关的重点股票。

盘中动态池必须标注时间；盘后以收盘快照为准。只有用户要求解释个股原因时，才进一步调用个股异动 skill。

### 机构调研

当前/最近交易日优先调用 `cn-institutional-survey-heat` 或 MX 调研记录查询。输出统计窗口、机构调研次数、股票 Top10 和行业 Top10，并保持机构接待对象口径一致。

历史请求优先读取公开历史调研热度数据，校验目标日期、调研开始日期和生成时间。若指定日期或字段不可用，原样说明“尚不支持该时间/数据”。

## 接口与限流

- 优先使用 MX 的聚合数据，不用逐股票、逐板块、逐日期循环请求。
- 使用备用接口时，同一个 host 连续请求超过 3 次要加入适度随机等待，避免紧密循环和高并发。
- 如果出现 HTTP 429/403/5xx、超时、DNS 失败、连接重置或非 JSON 响应，报告 host、接口类别和错误类型，并停止继续增加该 host 的请求量。
- 盘中数据只能表述为“截至某时刻的快照”；盘后数据必须校验目标交易日；周末和节假日使用最近交易日并明确说明。

## 依赖

- 题材强弱必需：`mx-zixuan`、`mx-xuangu`、`mx-search`。
- 资金流、涨停池和机构调研优先：`mx-data`。
- 机构调研协同：`cn-institutional-survey-heat`。
- 异动解释协同：`cn-stock-move-reason`、`macro-news-check`，仅在用户要求时调用。

## 本地缓存刷新

正常运行时，如果题材缓存缺失或超过 7 天，可以刷新本地快照：

```bash
python3 <cn-market-tape>/scripts/refresh_theme_assets_from_public_source.py
```

也可以从已确认可用的本地目录复制题材映射文件：

```bash
python3 skills/cn-market-tape/scripts/refresh_theme_assets.py /path/to/public-theme-source
```

脚本只校验并写入题材映射缓存；缓存、原始 API 响应、账户信息、完整自选股列表和运行输出不应提交到公开仓库。

## 安全边界

- `mx-zixuan` 只查询，不添加、删除或修改自选股。
- 不提交 `MX_APIKEY`、cookie、账户标识、个人持仓、自选股、私有研究资料或原始响应。
- 不把不同时间、不同统计窗口和不同源的数字拼成看似精确的结论。
- 本 skill 输出仅用于市场研究，不构成投资建议。
