# 题材/板块分时资金图 SOP

本文件规定 `cn-market-tape` 在用户要求“题材分时”“板块分时流入流出”“资金折线图”时的统一流程。它描述公开聚合数据的读取与绘图口径，不保存原始响应、用户账户数据或会话图表数据到仓库。

## 1. 先确认分析对象

- `行业板块`、`概念题材`和单只股票不是同一层级，必须按用户指定层级取数。
- “融资融券”先确认是 `融资融券` 概念板块，还是全市场融资融券账户的日度统计。后者不能伪装成分钟级板块资金曲线。
- 多个题材/板块要尽量使用同一来源、同一交易日和同一请求时点；若不能做到，报告时间差和不可比原因。

## 2. 解析并验证板块代码

1. 行业板块使用聚合板块列表的 `m:90+t:2`，概念板块使用 `m:90+t:3`。
2. 从返回的板块名称和代码中选择目标对象；不要根据记忆猜代码。
3. 分时接口返回后，再检查 `data.name` 是否与请求的板块名称精确一致。名称不一致、返回的是成分股列表或无法确认层级时，停止绘图并重新解析。
4. 首次验证过的代码可以在短时重复查询中复用，但每次分时响应仍要做名称复核；不能只凭上次代码沿用。

## 3. 读取分钟级序列

使用当前交易日的 1 分钟资金流接口，参数口径为 `klt=1`、`lmt=240`，字段为 `f51,f52,f53,f54,f55,f56`。字段含义：

| 字段 | 含义 | 绘图处理 |
| --- | --- | --- |
| `f51` | 时间戳 | 原样保留并显示最新时间 |
| `f52` | 主力净流入累计值 | 主曲线，解析元后换算亿元 |
| `f53` | 小单净流入 | 仅在用户要求拆分时展示 |
| `f54` | 中单净流入 | 仅在用户要求拆分时展示 |
| `f55` | 大单净流入 | 与 `f56` 做一致性检查 |
| `f56` | 超大单净流入 | 与 `f55` 做一致性检查 |

在绘图前检查 `f52 ≈ f55 + f56`（允许数据源四舍五入误差）。接口原始值按元解析后再换算为亿元，不能把元直接当作亿元。

保留接口原样时间序列和交易时段缺口：午间休市、接口缺点、从 09:31 开始等都不插值、不补点。记录两个时间：资金榜单的更新时间，以及分钟序列的最新点。分钟接口较慢时，图标题/状态行明确写“分时最新点落后于榜单”。

## 4. 统一绘图规范

- 默认绘制 `f52` 累计主力净流入曲线，而不是每分钟增量。
- 纵轴统一为“主力净流入（亿元）”，绘制清晰的 0 轴。
- 标注盘中低点、高点、零轴穿越和最新点；午间缺口保持为空白。
- 图例、标题或副标题写明板块名称、交易日、最新时间、来源、单位和“累计值”。
- 同时查看多个板块时共享横轴；如果金额尺度差异明显，使用分面/小 multiples，不能为了叠加而去掉单位或无说明地归一化。
- 如果用户明确要求“每分钟流入流出/增量”，才对相邻 `f52` 做差分；另画或另表展示，并明确标为“每分钟增量”，不能和累计曲线混称。
- 图表后仍给出最新快照表，至少列板块、分时最新时间、主力累计净额；图表不替代 Top10 或当前快照。

## 5. 解读与异常处理

用中性结构描述曲线：持续流出、早盘流出后修复、早盘流入后派发、或高位双向分歧。最新值转正但盘中曾深度为负时，只称为“修复路径”，不直接称为全天净流入趋势。

若排名接口或分时接口返回空数据、非 JSON、HTTP 429/403/5xx、超时、DNS 失败或连接重置，记录 host、接口族、错误类型和已尝试次数，停止继续加大该 host 的请求量。可使用已验证的同口径快照表；无法提供可靠曲线时明确说明“分时图不可用”，不凭排名快照补画分钟数据。

多次查询的比较只在交易日、来源、单位、板块层级、统计窗口和时间点一致时计算变动。若只能比较各自最新点，表中同时列出两个时间并标注时间不一致。
